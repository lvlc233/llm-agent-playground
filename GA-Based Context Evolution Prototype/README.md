# 基于遗传算法的上下文进化原型

## 核心概念:
上下文(Context)
Context <- Sequence(Context Chunks)
Fllm(Sequence(Context Chunks)) -> new Context
Fllm(Context) -> Sequence(Context Chunks)

一个上下文应该被描述为有多个有序的上下文块的集合。
将一组上下文块输入给LLM，LLM会根据上下文块的内容生成新的上下文信息
并通过LLM将这一组的上下文切割成多个不同的有含义的块

遗传算子: Context Chunk
对于一个目标Target,一个最优的上下文应该通过 Fllm(Sequence(Context Chunks)) -> new Context,并使得该New Context与Target的平均相似度最高。
所以两个部分,第一:相似,第二:平均(要求该上下文不能一两次出现相同的上下文块,而是应该总是得到这样子的结果)

遗传优化部分: 评估 当前 context chunk 与 target 的贡献度。
流程: 
生成目标:->转化为模型上下文->对该上下文进行切块以分为多个子目标
对多个子目标独立生成一组上下文块,每个上下文块都放入一个池中,该池就作为运行时候的遗传因子库,随机的组合任意的一组上下文块,并输入给LLM,LLM会根据上下文块的内容生成新的上下文信息,并对比新的上下文信息与目标的相似度。持续若干论后,取得相似度最高的上下文(非上下文块)若干组上下文进行统计分析,并根据上下文块出现的数量来重新计算每个上下文的突变概率。出现次数越低的块其突变概率越大,并设定好最大最小的突变概率,之后随机让剩余的(没有被淘汰的)上下文块进行chunk交换,生成新的有序上下文chunk集合,并准备下一轮
